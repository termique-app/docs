Termique's sync backend is open source and designed to be self-hosted. Your hosts, groups, snippets, and encrypted credentials are stored on infrastructure you control.

## Stack

The API backend is a Node.js service:

| Component | Technology |
|-----------|-----------|
| Framework | Hono 4 |
| ORM | Drizzle ORM |
| Database | PostgreSQL (+ pgvector for AI memory) |
| Auth | Firebase Admin SDK |
| Container | Docker |

Source: [`termique-app/termique-api`](https://github.com/termique-app/termique-api)

## Prerequisites

- PostgreSQL 15+ (with pgvector extension for AI features)
- Docker (recommended) or Node.js 20+
- A Firebase project (for user authentication)
- A Cloudflare Workers AI account (optional — for the hosted AI assistant)

## Environment variables

Create a `.env` file:

```env
PORT=8787
DATABASE_URL=postgresql://user:password@localhost:5432/termique
FIREBASE_PROJECT_ID=your-project-id
FIREBASE_SERVICE_ACCOUNT={"type":"service_account",...}
SHARING_SECRET=at-least-32-chars-random-string
ADMIN_SECRET_KEY=your-admin-secret
# Optional — for hosted AI assistant:
CLOUDFLARE_ACCOUNT_ID=...
CLOUDFLARE_AI_API_KEY=...
```

`SHARING_SECRET` must be at least 32 characters. The server will refuse to start without it.

## Run with Docker

```bash
docker run -d \
  --name termique-api \
  -p 8787:8787 \
  --env-file .env \
  ghcr.io/termique-app/api:latest
```

## Run without Docker

```bash
git clone https://github.com/termique-app/termique-api
cd termique-api
npm install
npm run dev     # development
npm start       # production
```

## Point the app at your backend

In Termique desktop, open **Settings → Sync** and set the **API URL** to your backend address, e.g. `https://api.your-domain.com`.

On first sync, the app will register your user and push local data to the backend.

## Health check

```bash
curl https://api.your-domain.com/health
# → {"ok":true}
```
